# 勤怠アプリ — 要求仕様書

## 概要

社員の出退勤を Web 画面で打刻・管理する勤怠管理アプリケーション。

- 対象ユーザー: 社員（一般）+ 管理者
- 想定規模: 数十人
- 勤務形態: フレックスタイム制（所定労働時間 8時間）

## ユーザーストーリー

### 打刻

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-1 | 社員として、出勤ボタンを押して出勤を記録したい | 当日未打刻の状態で出勤ボタンを押すと、現在時刻が出勤時刻として記録される |
| US-2 | 社員として、退勤ボタンを押して退勤を記録したい | 出勤済み・未退勤の状態で退勤ボタンを押すと、現在時刻が退勤時刻として記録される |
| US-3 | 社員として、打刻忘れを自分で修正したい | 本人が過去の打刻を編集できる |
| US-4 | 管理者として、部下の打刻を修正したい | 管理者が任意の社員の打刻を編集できる |

### 勤怠履歴・集計

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-5 | 社員として、自分の月次勤怠一覧を確認したい | 指定月の日別出勤・退勤・勤務時間・残業時間が一覧表示される |
| US-6 | 管理者として、部下の月次勤怠一覧を確認したい | 指定社員・指定月の勤怠一覧が表示される |
| US-7 | 社員として、月の合計勤務時間と残業時間を確認したい | 月次集計（総勤務時間、総残業時間）が表示される |

### 社員管理

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-8 | 管理者として、社員を登録したい | 社員名・メールアドレス等を入力して社員を作成できる。社員IDは自動採番 |
| US-9 | 管理者として、社員情報を編集・削除したい | 既存社員の情報を更新・論理削除できる |

### 認証

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-10 | ユーザーとして、ID/パスワードでログインしたい | 正しい認証情報でログインし、セッションが開始される |
| US-11 | 管理者として、管理者専用機能にのみアクセスできるよう制御したい | 一般社員は管理者機能にアクセスできない |

## ビジネスルール

- **休憩時間**: 自動控除（勤務6時間超で45分、8時間超で60分 — 労基法準拠）
- **残業時間**: 1日の実勤務時間 − 所定労働時間（8時間）。マイナスは0とする
- **フレックス**: コアタイム等の制約は設けない（打刻時刻の制限なし）

## スコープ外（将来対応）

- 有給休暇・休日申請機能
- CSV エクスポート
- 位置情報付き打刻
- SSO / 外部認証連携

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| Frontend | TypeScript / Next.js |
| Backend | Java / Spring Boot |
| DB | H2（開発環境） |

## 権限モデル

| ロール | できること |
|--------|-----------|
| 一般社員 | 自分の打刻、自分の打刻修正、自分の勤怠履歴・集計閲覧 |
| 上司 | 上記すべて + 部下の勤怠一覧・集計閲覧 + 部下の打刻修正 |
| 管理者 | 上記すべて + 全社員の勤怠閲覧・打刻修正 + 社員管理 |
